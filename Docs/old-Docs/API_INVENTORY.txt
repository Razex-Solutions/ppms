PPMS BACKEND API INVENTORY
Generated from current backend route modules

Purpose
- This file is a plain-text overview of the PPMS backend APIs.
- It lists what modules exist, what they generally do, and which endpoints are currently available.
- Most routes are protected and require login unless noted otherwise.

Base Notes
- Root health and app metadata live outside these module routers.
- Authentication starts with /auth/login.
- Most modules are role-scoped and station/organization scoped by backend rules.

==================================================
1. AUTH
==================================================
What it does
- Login, token refresh, session visibility, logout, password changes, and admin password reset.

Endpoints
- POST /auth/login
- GET /auth/me
- POST /auth/refresh
- GET /auth/sessions
- POST /auth/logout
- POST /auth/logout-all
- POST /auth/change-password
- POST /auth/admin-reset-password/{user_id}
